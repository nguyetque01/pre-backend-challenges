# Mục lục Bài tập Pre-Backend

## Giới thiệu
Dưới đây là danh sách các bài tập (challenges) trong khóa học Pre-Backend. Mỗi bài tập được thiết kế để giúp bạn áp dụng kiến thức chuyên sâu về Node.js, PostgreSQL, MongoDB và các chủ đề backend tổng quát. Hãy hoàn thành từng bài tập theo thứ tự để xây dựng nền tảng vững chắc.

### Tài liệu Lý thuyết
- [Node.js và Express.js Guide](./node/node-express-guide.md) - Lý thuyết và bài tập về Node.js, Express, thiết kế source, tư duy backend.
- [SQL Queries Types](./sql/sql-queries-types.md) - Các loại truy vấn SQL, NoSQL và bài tập thực hành.

## Danh sách Bài tập

### 1. [Challenge 1: Hello World với Node.js](./challenges/challenge-1.md)
- Mục tiêu: Làm quen với Node.js cơ bản.
- Nội dung: Tạo một script đơn giản in ra "Hello World" và chạy nó.

### 2. [Challenge 2: Xây dựng HTTP Server](./challenges/challenge-2.md)
- Mục tiêu: Hiểu về server-side với Node.js.
- Nội dung: Sử dụng Express.js để tạo một server HTTP đơn giản trả về JSON.

### 3. [Challenge 3: Cơ bản PostgreSQL Queries](./challenges/challenge-3.md)
- Mục tiêu: Làm việc với cơ sở dữ liệu PostgreSQL.
- Nội dung: Viết các câu lệnh SQL để truy vấn dữ liệu từ bảng mẫu trong PostgreSQL.

### 4. [Challenge 4: Kết nối Database với Node.js](./challenges/challenge-4.md)
- Mục tiêu: Kết hợp Node.js với PostgreSQL.
- Nội dung: Sử dụng pg để kết nối và truy vấn database từ Node.js.

### 5. [Challenge 5: Giới thiệu MongoDB](./challenges/challenge-5.md)
- Mục tiêu: Làm quen với NoSQL.
- Nội dung: Thực hiện CRUD operations trên MongoDB.

### 6. [Challenge 6: API với MongoDB](./challenges/challenge-6.md)
- Mục tiêu: Xây dựng API sử dụng NoSQL.
- Nội dung: Tạo RESTful API với Express và Mongoose để quản lý dữ liệu.

### 7. [Challenge 7: Authentication với JWT](./challenges/challenge-7.md)
- Mục tiêu: Triển khai authentication cơ bản.
- Nội dung: Sử dụng JWT để bảo vệ routes trong Express app.

### 8. [Challenge 8: Testing với Jest](./challenges/challenge-8.md)
- Mục tiêu: Học testing.
- Nội dung: Viết unit và integration tests cho Node.js app.

### 9. [Challenge 9: Deployment với Docker](./challenges/challenge-9.md)
- Mục tiêu: Containerize ứng dụng.
- Nội dung: Sử dụng Docker để deploy Node.js app.

### 10. [Challenge 10: Caching với Redis](./challenges/challenge-10.md)
- Mục tiêu: Cải thiện performance.
- Nội dung: Implement caching cho API sử dụng Redis.

### 11. [Challenge 11: GraphQL API](./challenges/challenge-11.md)
- Mục tiêu: Xây dựng API linh hoạt.
- Nội dung: Tạo GraphQL API với Apollo Server.

### 12. [Challenge 12: Microservices Basics](./challenges/challenge-12.md)
- Mục tiêu: Hiểu microservices.
- Nội dung: Xây dựng hệ thống với nhiều services giao tiếp nhau.

### 13. [Challenge 13: Error Handling và Logging](./challenges/challenge-13.md)
- Mục tiêu: Xử lý lỗi và logging.
- Nội dung: Implement error handling và logging với Winston.

### 14. [Challenge 14: Security Best Practices](./challenges/challenge-14.md)
- Mục tiêu: Áp dụng bảo mật.
- Nội dung: Sử dụng Helmet, CORS, rate limiting.

### 15. [Challenge 15: Environment Variables và Configuration](./challenges/challenge-15.md)
- Mục tiêu: Quản lý config.
- Nội dung: Sử dụng dotenv cho environment variables.

### 16. [Challenge 16: API Versioning](./challenges/challenge-16.md)
- Mục tiêu: Version APIs.
- Nội dung: Implement versioning cho backward compatibility.

### 17. [Challenge 17: File Upload với Multer](./challenges/challenge-17.md)
- Mục tiêu: Handle file uploads.
- Nội dung: Upload files với Multer.

### 18. [Challenge 18: Email Sending với Nodemailer](./challenges/challenge-18.md)
- Mục tiêu: Gửi emails.
- Nội dung: Integrate email sending vào app.

### 19. [Challenge 19: Scheduling Tasks với Node-Cron](./challenges/challenge-19.md)
- Mục tiêu: Schedule tasks.
- Nội dung: Chạy recurring tasks với cron.

### 20. [Challenge 20: WebSockets với Socket.io](./challenges/challenge-20.md)
- Mục tiêu: Real-time communication.
- Nội dung: Implement WebSockets cho chat hoặc notifications.

## Hướng dẫn Chung
- Mỗi challenge có file riêng với mô tả chi tiết, yêu cầu, và gợi ý giải quyết.
- Sử dụng Git để commit từng bước hoàn thành.
- Nếu gặp khó khăn, tham khảo tài liệu trong README.md hoặc hỏi giảng viên.
